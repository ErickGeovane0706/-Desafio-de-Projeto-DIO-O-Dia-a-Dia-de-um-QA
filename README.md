## 📝 Sobre o Projeto
Este repositório contém a entrega do desafio de projeto **"O dia a dia de um QA: A Prática de Testes Manuais Funcionais"**, proposto pela [DIO (Digital Innovation One)](https://www.dio.me/). O objetivo do projeto é aplicar os conceitos de qualidade de software através da criação e documentação de testes para um e-commerce fictício chamado **SwagLabs Shopping**, simulando um ambiente real de metodologias ágeis (Scrum/Kanban).

## 🚀 Entregáveis (Artefatos Gerados)
O projeto foi desenvolvido utilizando a ferramenta Jira e o plugin Zephyr Scale. Todos os artefatos visuais estão documentados através das imagens em anexo neste repositório:


1. **Fluxo de Trabalho e Ciclo de Vida do Bug** (<img width="1876" height="927" alt="print1" src="https://github.com/user-attachments/assets/d8bb6b68-ed6a-4f32-9b21-cbc3d501a708" />)
   - Configuração do board no Jira com as colunas (To Do, In Progress, Ready for QA, Done) simulando o fluxo de desenvolvimento de uma funcionalidade e o caminho de resolução de bugs.

2. **User Stories (Histórias de Usuário)** (<img width="1875" height="972" alt="print2" src="https://github.com/user-attachments/assets/aa0d0690-3d9c-4eae-8c5f-b1b315e30598"/>
)
   - Criação de Épico e Histórias de Usuário mapeadas no Backlog do Jira, estruturadas com a narrativa BDD (Como... Desejo... Para...) e seus respectivos Critérios de Aceite.

3. **Mapa Mental de Cenários de Teste** (<img width="1643" height="927" alt="print3" src="https://github.com/user-attachments/assets/1e4843e6-28cf-4be1-b5c1-8012231fc424" />)
   - Mapeamento visual exploratório derivando os cenários possíveis (Caminho Feliz e Caminhos de Erro) para a funcionalidade de Login.

4. **Casos de Teste (Step-by-Step e BDD)** (<img width="1877" height="929" alt="print4" src="https://github.com/user-attachments/assets/e7c0aa72-00af-45ab-bb38-150741ae7f16" />)
   - Criação de casos de teste no **Zephyr Scale** utilizando técnicas distintas:
     - 2 Casos baseados em *Step-by-Step* (Passo a Passo) para validação de layout e regras de negócio.
     - 2 Casos baseados em *BDD (Gherkin)* utilizando *Given-When-Then* para cenários de comportamento.

5. **Ciclo de Testes (Test Cycle)** (<img width="1879" height="930" alt="print5" src="https://github.com/user-attachments/assets/a8fadfe8-944b-4b58-bde2-dfdc0ae0a149" />)
   - Agrupamento dos 4 casos de teste desenvolvidos em um ciclo de testes formal para garantir a rastreabilidade e organização da execução na Sprint 1 (Autenticação).

## 🛠️ Ferramentas Utilizadas
* **Jira Software:** Gestão ágil e fluxo de trabalho.
* **Zephyr Scale:** Gerenciamento de Casos e Ciclos de Teste.
* **HTML/CSS:** Criação estruturada do mapa mental de cenários.
